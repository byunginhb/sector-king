## 0. 사전 조사 — 기존 자산과 중복/충돌 지점 (반드시 먼저 읽을 것)

구현 전에 다음 3개 기존 자산을 반드시 인지해야 한다. 새 페이지는 이들과 **역할을 나누고**, 일부는 **정직하게 바로잡는다.**

| 기존 자산 | 위치 | 성격 | 새 `/guide`와의 관계 |
|---|---|---|---|
| `/methodology` | `app/methodology/page.tsx` | 산식·가중치·EMA·선정기준·데이터품질 **상세** (전문 톤) | 중복 금지. 새 페이지에서 "더 깊이 알고 싶다면" 링크로 연결 |
| 용어 사전 (Glossary) | `components/onboarding/glossary-data.ts` + `glossary-panel.tsx` (모달) | 11개 용어 짧은 정의, 검색·카테고리 탭 모달 | 데이터 재사용 가능. 단 아래 **충돌 2건** 수정 필요 |
| 온보딩 | `components/onboarding/*`, `app/me/onboarding/page.tsx` | 투어·환영모달(앱 사용법) / `/me/onboarding`(관심 산업 선택, 로그인 필요) | 가이드는 "개념 이해", 온보딩은 "행동 유도". 상호 링크 |

### ⚠ 발견된 정합성 충돌 (가이드 작성 시 반드시 정정)

1. **CRITICAL — glossary의 자금흐름 정의가 틀림.**
   `glossary-data.ts`의 `inflow`/`outflow`는 "섹터의 **시가총액**이 증가/감소한 상태"로 설명한다.
   그러나 실제 산식(`lib/money-flow-helpers.ts:176-178`)은 `typicalPrice((H+L+C)/3) × volume`의 **거래대금 누적**이다(USD 변환 후). 시가총액 증감과 무관하다.
   → 새 가이드는 "자금 흐름 = **거래대금 기반 활동량 추정**이며, 실제 순매수 자금이 아니다"로 정직하게 정의. glossary 데이터도 동일하게 수정 권장(구현 단계 협업: audit-display.md).

2. **MEDIUM — money-flow UI 라벨 과대해석.**
   `flow-card.tsx:181` "유입 ↑ / 유출 ↓", `flow-summary.tsx` "총 유입 / 총 유출 / 순 유입".
   _context.md S4: 모든 거래는 매수=매도라 "순유입"은 이론상 측정 불가. 라벨은 그대로 두되, 가이드에서 "이 화살표는 자금이 실제로 들어온 게 아니라 **거래가 활발해진 방향**을 뜻한다"고 반드시 풀어 설명.

> 위 정정 문구의 **최종 표현**은 병렬 진행 중 `audit-theory.md` 결과를 구현 단계에서 반영한다. 본 기획에서는 자리표시자(아래 §4 "한계 고지")로 구조만 확정한다.

## 1. 포지셔닝 — `/guide` vs `/methodology`

### URL 제안: `/guide`
- 코딩스타일 규칙 준수: 경로 파라미터 없음, 정적 단일 세그먼트(영문 소문자). UUID/숫자 규칙은 동적 파라미터 대상이므로 무관.
- 대안 검토: `/help`(도움말 톤·HelpButton과 혼동), `/about`(회사 소개 뉘앙스). → **`/guide`** 채택("사용 안내서" 의미가 가장 명확).

### 역할 분담표

| 항목 | `/guide` (신규, 완전 초보) | `/methodology` (기존, 상세) |
|---|---|---|
| 독자 | 처음 온 사람, 비전문가 | 산식 검증·신뢰성 확인하려는 사용자 |
| 질문 | "이게 뭐고 어떻게 보나요?" | "점수가 정확히 어떻게 계산되나요?" |
| 톤 | 비유·예시 중심, 쉬운 말 | 공식·가중치·정규화 범위 |
| 패권 점수 | "4가지로 회사 힘을 0~100점 채점" 한 문장 + 비유 | scale 35/growth 30/profit 20/sentiment 15, 클램핑·중앙값 폴백 |
| 자금 흐름 | "거래가 활발한 방향, 실제 순유입 아님" | (현재 methodology에 자금흐름 섹션 없음 — 보완 후보) |
| 한계 고지 | 쉬운 말로 "주의사항" | 면책·데이터품질 정밀 |
| 연결 | 각 항목 끝에 "자세한 산식 →`/methodology#앵커`" | — |

→ **단방향 깊이 연결**: 가이드(얕음) → 메서돌로지(깊음). 푸터에는 둘 다 노출.

## 2. 콘텐츠 구조 (네 묶음)

페이지를 4개 큰 섹션으로 구성. 앵커 목차로 점프.

### 섹션 A — "이 서비스가 뭔가요" (`#what`)
- **3계층 비유**: 산업(예: 테크) → 섹터(반도체) → 종목(엔비디아). "백화점 → 매장 → 상품" 비유.
- **시장 범위**: 미국·한국 상장 종목만. 한국 종목 가격/시총도 전부 **달러($)로 환산해 보여줌**.
- **갱신 주기**: 하루 한 번(매일 00:00 KST). 실시간 아님 — 1문장 미리 고지(자세한 건 섹션 D).
- 시각: 3계층 다이어그램(기존 `DataPipeline` 스타일의 단순 박스 3개 + 화살표, lucide `ChevronRight`).

### 섹션 B — "숫자 사전" (`#numbers`) ★ 핵심
각 항목 공통 포맷 카드:
```
[아이콘] 용어 (영문)
한 줄 정의
└ 이렇게 읽으세요: (해석 가이드)
└ 비유/예시: (구체 숫자 예)
└ [직접 보러 가기 →]  (실제 페이지 링크)  · [자세히 →](/methodology#앵커)
```

다룰 항목(순서):

| # | 항목 | 한 줄 정의 | 이렇게 읽으세요 | 비유/예시 | 보러 가기 |
|---|---|---|---|---|---|
| 1 | 패권 점수 (0~100) | 회사의 시장 지배력을 4가지로 채점한 종합 점수 | 높을수록 그 섹터의 "왕"에 가깝다 | 학생 성적표(국·영·수·과 합산) | `/[ind]` 대시보드, `/[ind]/statistics` |
| 1a | └ 4개 축 | 규모35·성장30·수익성20·시장평가15 | 같은 90점이라도 큰 회사형 vs 빠르게 크는 회사형 다름 | 4과목 비중 다른 시험 | `/methodology#scoring` |
| 2 | 시가총액 ($) | 주가 × 발행주식수 = 회사 전체 가격표 | 회사 "덩치". 한국 종목도 달러로 환산 표시 | T=1조달러, B=10억달러 | 모든 페이지 |
| 3 | 등락률 (%) | 어제 대비 주가 변화율 | +면 올랐고 −면 내렸다 | +2.3% = 100→102.3 | `/[ind]/price-changes` |
| 4 | 52주 위치 | 1년 최고/최저가 사이 현재 위치 | 100%에 가까우면 1년 신고가권 | 0~100 막대의 현재 점 | `/[ind]/statistics`, `/stock/[t]` |
| 5 | 섹터 순위 | 섹터 안 패권 점수 등수 | 1위 = 그 분야 대장주 | 반 1등 | 대시보드, hegemony-map |
| 6 | 시총 점유율 (%) | 섹터 전체 시총 중 이 회사 몫 | 높을수록 섹터 장악 | 시장 케이크 한 조각 | hegemony-map, `/stock/[t]` |
| 7 | 자금 흐름 | **거래대금 기반 활동량 추정** (순유입 아님) | "유입"=거래 활발해진 방향일 뿐, 실제 돈이 들어온 게 아님 | 가게 손님 북적임 ≠ 매출 순증 | `/[ind]/money-flow` |
| 8 | PER / PEG | 이익 대비 주가가 비싼지 (성장 감안=PEG) | 낮을수록 싸다(단, 업종별 기준 다름) | PER 10 = 10년치 이익 = 주가 | `/[ind]/statistics`, `/stock/[t]` |
| 9 | ROE (%) | 자기자본으로 얼마나 벌었나 | 높을수록 돈 굴리기 잘함 | 100만원으로 15만원 = ROE 15% | `/stock/[t]` |
| 10 | 목표주가 상승여력 | 애널리스트 목표가 vs 현재가 차이(%) | +면 더 오른다는 기대 | 현재 $100, 목표 $120 = +20% | `/stock/[t]` |

> 항목 7·8 등 데이터 결측 빈번 항목은 §4 한계 고지로 연결. PER/PBR 명칭은 glossary가 sentiment를 "PER/PBR"로 설명하나 실제 산식(`methodology`)은 목표주가 괴리율 기반 — 가이드에서는 "시장평가 점수 = 애널리스트 기대(목표주가) 중심"으로 정정해 일관화.

### 섹션 C — "화면별 읽는 법" (`#screens`)
각 화면: 무엇을 보는 곳(1~2문장) + 대표 요소 1~2개 설명 + "직접 보기 →" 링크.

| 화면 | 무엇을 보는 곳 | 대표 요소 |
|---|---|---|
| 대시보드 `/[ind]` | 산업 전체를 한눈에, 섹터별 패권 지도로 드릴다운 | 산업 카드, 14일 시총 추세 스파크라인 |
| 자금 흐름 `/[ind]/money-flow` | 어느 섹터에 거래가 몰리는지 (활동량) | 유입/유출 카드(색), MFI 막대 — "방향일 뿐" 주석 |
| 등락 `/[ind]/price-changes` | 오늘 누가 많이 오르고 내렸나 | 등락율 표·차트, 상위 20개 |
| 패권 지도 `/[ind]` (map) / hegemony-map | 섹터별 점유율·순위를 시각적으로 | 점유율 비중, 패권 점수 |
| 종목 상세 `/stock/[t]` | 한 회사 깊이 파보기(추이·peer·밸류에이션·시그널) | 점수 74일 추이, peer 비교, 시그널 룰 |

### 섹션 D — "정직한 한계 고지" (`#limits`) ★ 유료 신뢰 핵심
> 모든 문구는 audit-theory.md 확정본을 구현 시 반영. 아래는 **구조 + 자리표시자**.

- 데이터 출처: Yahoo Finance(yfinance). 3자 데이터라 지연·오류 가능. _[자리표시자: 정확 문구]_
- 갱신 주기: 1일 1회(00:00 KST). 장중 변동·프리마켓 미반영.
- 환율 환산: 한국 종목 KRW→USD는 **고정 환율** 사용(현재 1450 가정). 실시간 환율과 차이. _[자리표시자: S1/S2 결론 — SoT 일원화 후 실제 값/방식]_
- 결측 처리: 데이터 없는 지표는 중간값(50%)으로 대체 → 점수 정확도 하락 가능. "데이터 제한적" 배지 안내.
- 스냅샷 시점: 한·미 장마감 시각이 달라 같은 날짜라도 한쪽은 장중가일 수 있음. _[자리표시자: S7]_
- 자금 흐름의 의미: **실제 순유입 아님**, 거래대금 추정. (섹션 B-7과 교차링크)
- 투자 권유 아님: 정보 제공 목적, 투자 책임은 이용자. (methodology 면책과 동일 톤)

각 항목 끝: "더 자세히 →`/methodology#disclaimer`".

## 3. UX / 구현 설계

### 인터랙션 (과하지 않게)
- **앵커 목차** 1개(섹션 A~D 4개 + 숫자사전 하위 점프). methodology와 동일한 `<nav aria-label="목차">` 패턴 재사용.
- 숫자 사전: **아코디언 불필요** — 각 항목이 짧으므로 카드 그리드로 항상 펼쳐 노출(스캔성↑). 검색도 불필요(10개 수준, 모달 glossary가 검색 담당).
- 상단 1줄 "현재 보고 있는 화면이 헷갈리면 여기" 식의 sticky 목차는 지양(정적 페이지, 길이 적당).
- 각 숫자 항목·화면 항목에 **실제 페이지 직접 링크**. 산업 의존 경로는 기본 산업(예 `/tech/...`) 또는 대시보드(`/`)로 안전 연결.

### 진입점
- **GlobalTopBar `NAV_ITEMS`에 "이용 안내" 추가** (`{ href: '/guide', label: '이용 안내', icon: BookOpen }`). 데스크탑 inline + 모바일 Sheet 양쪽 자동 노출(기존 map 구조 그대로).
- **푸터 nav에 "이용 안내" 링크 추가** (방법론 앞).
- **HelpButton / 환영모달**: "처음이신가요? 이용 안내 보기 →`/guide`" 링크 1개 추가 검토(온보딩과 보완 관계, 중복 강요 X).
- `/me/onboarding`(로그인 필요·관심산업 선택)과는 별개 — 가이드는 비로그인 누구나. 온보딩 완료 화면에서 가이드로 안내 가능.

### 컴포넌트 분할 (200~400줄, 기능별)
```
app/guide/page.tsx                       # Server Component, generateMetadata + JSON-LD, 섹션 조립 (~120줄)
components/guide/guide-toc.tsx           # 앵커 목차 (~50줄)
components/guide/guide-section.tsx       # 섹션 래퍼(제목+id+scroll-mt) 재사용 (~30줄)
components/guide/service-intro.tsx       # 섹션 A: 3계층 다이어그램 (~120줄)
components/guide/number-glossary.tsx     # 섹션 B: 숫자 사전 카드 그리드 (~80줄, 데이터 분리)
components/guide/number-glossary-data.ts # 섹션 B 항목 데이터(term/정의/읽기/비유/링크) (~120줄)
components/guide/screen-guide.tsx        # 섹션 C: 화면별 (~100줄)
components/guide/honest-limits.tsx       # 섹션 D: 한계 고지 (~90줄)
```
- 재사용: `components/ui/card.tsx`(Card/CardHeader/...), `badge.tsx`(Badge), lucide 아이콘. methodology의 로컬 `Card`/`SectionHeading` 대신 **공용 ui/card** 사용 권장(일관성).
- 데이터-뷰 분리: 사전 항목은 `*-data.ts`에 두어 audit 결과 반영 시 데이터만 수정.

### 렌더링 / SEO
- **Server Component 권장**: 콘텐츠 정적, 인터랙션은 앵커 점프(href)뿐 → `'use client'` 불필요. 빠르고 SEO 유리.
- `generateMetadata`(또는 정적 `metadata`): title "이용 안내", 초보자 대상 description.
- **FAQ JSON-LD 적용 검토**: 숫자 사전을 Q("패권 점수가 뭔가요?")/A 형태로 `FAQPage` schema.org JSON-LD 주입 → Google 생성형/리치결과 노출 기대(seo 관점 권장). `<script type="application/ld+json">`로 Server Component에서 직접 렌더.
- `app/sitemap.ts` `staticPages`에 `/guide` 추가(priority 0.7, changeFrequency monthly — 정적 콘텐츠).

### a11y / 스타일 규칙
- **이모지 금지** — 모든 아이콘 lucide-react(BookOpen, Layers, DollarSign, TrendingUp, BarChart3, Crown, PieChart, Activity, AlertTriangle 등).
- 목차 `<nav aria-label="목차">`, 섹션 `<section>` + `<h2 id scroll-mt-20>`.
- 링크는 의미있는 텍스트("자금 흐름 화면 보기"), 아이콘에 `aria-hidden`.
- 반응형 표준 헤더 패턴(`flex flex-col sm:flex-row sm:items-center sm:justify-between gap-3`)은 GlobalTopBar가 담당 — 본문은 `container mx-auto px-4 py-8 max-w-3xl`(methodology와 동일).
- 한계 고지 박스: `bg-warning-bg border-warning/30` + `AlertTriangle`(기존 methodology 톤 재사용).
