# 섹터킹 고도화 — GitHub 이슈 등록용 리스트

회의 아이디어 중 **코드/데이터로 구현 가능한 개선·고도화 건만** 필터링. 각 항목은 깃헙 이슈로 바로 복붙 가능하도록 작성.

> 라벨 표기: `유형` / `우선순위(P1~P3)` / `난이도(S·M·L)`
> 전략·비즈니스 영역(타겟·수익화·정책 등 "고민 건")은 맨 아래 별도 정리.

---

## A. 퀵윈 (UX·버그 — 작고 즉시 가능)

### 1. 다크모드 가독성 개선 (PC)
- **라벨**: bug, ui / P1 / S
- **목적**: 모바일은 정상이나 PC 다크모드에서 일부 요소 가독성 저하. 금융권 유저가 다크 선호하므로 중요.
- **상세**: PC 뷰포트에서 다크모드 대비(contrast) 점검 → 텍스트/카드/차트 색상 토큰 보정. 디자인 시스템(블룸버그 기반) 색상 변수 단위로 수정.
- **수용 기준**: 주요 페이지(대시보드/money-flow/price-changes/statistics) PC 다크모드에서 텍스트·라인 가독성 확보, 라이트모드 무회귀.

### 2. 30일 필터 + 종목 클릭 시 십자가(크로스헤어) 깨짐 버그
- **라벨**: bug / P2 / S
- **목적**: 30일 토글 후 임의 종목 클릭 시 차트 십자선이 비정상 표시.
- **상세**: 재현 → 차트 인터랙션(crosshair) 좌표/툴팁 로직 점검. recharts 기준 활성 데이터 인덱스 처리 확인.
- **수용 기준**: 모든 기간 토글에서 종목 클릭 시 십자선/툴팁 정상 동작.

### 3. 모션(애니메이션) on/off 토글
- **라벨**: enhancement, ui / P3 / S
- **목적**: 움직이는 마켓 티커가 눈에 띄지만 정신없다는 피드백. 사용자가 끌 수 있게.
- **상세**: 상단 티커/역동 요소에 모션 토글 추가, 설정 localStorage 저장(테마 토글 패턴 재사용).
- **수용 기준**: 토글 off 시 애니메이션 정지, 새로고침 후 설정 유지.

## B. 기존 기능 고도화

### 4. 뉴스 출처(소스) 명시
- **라벨**: enhancement, data / P1 / M
- **목적**: 현재 뉴스/요약에 출처가 없어 신뢰성 부족. "어디까지가 사실인가" 검증 불가.
- **상세**: 뉴스 수집 파이프라인에서 원문 출처(언론사·URL) 메타데이터 보존 → 카드/요약에 출처 배지 표시. "~에 따르면" 류 인용 출처 파싱.
- **수용 기준**: 노출되는 뉴스/요약 항목에 출처 링크 또는 출처명 표기.

### 5. 사/마라 신호 고도화 — 근거·출처·기간 구분
- **라벨**: enhancement / P1 / M
- **목적**: 회의에서 가장 임팩트 컸던 요소("사라/마라"). 단, 막연한 판단이 아니라 근거를 제시해야 신뢰.
- **상세**: LLM 판단 결과에 (1) 판단 근거, (2) 근거 출처, (3) 단기/장기 시계 구분("당장 사지 마라" vs "묵힐 거면 사라")을 함께 출력. 투자 권유 아닌 정보 제공 면책 문구 유지.
- **수용 기준**: 각 신호에 근거 텍스트 + 출처 + 기간 구분 표시. 면책 고지 노출.

## C. 신규 데이터·기능

### 6. 키워드별 맞춤 뉴스 수집
- **라벨**: new-feature, data / P2 / M
- **목적**: 산업/관심 키워드 선택 시 해당 키워드 중심 뉴스 필터·수집.
- **상세**: 사용자 키워드 입력 → 수집 단계 또는 조회 단계에서 키워드 매칭 필터. 산업 카테고리와 연동.
- **수용 기준**: 키워드 지정 시 관련 뉴스만 필터링되어 노출.

### 7. 시장 규모 시각화 (매출·시총·포워드 가이던스)
- **라벨**: new-feature, viz / P2 / M
- **목적**: 돈의 흐름 외에 "이 시장이 얼마나 커질지"를 시각적으로. 섹터별 매출·시총·성장 전망.
- **상세**: 섹터/산업별 시장 규모(매출액 합산·시총)와 포워드 가이던스(애널리스트 성장 전망) 그래프 추가. 카테고리별 시총 비율. ⚠️ 블룸버그식 검정+빨강 네모는 진부하므로 차별화된 비주얼.
- **수용 기준**: 섹터 선택 시 시장 규모/성장 전망 차트 노출. (※ 시총 합산 시 `toUsd` 후 집계)

### 8. FRED 매크로 데이터 기반 국면(regime) 분석
- **라벨**: new-feature, data / P1 / L
- **목적**: 타겟층(매크로 투자자)과 가장 정합. CPI 등 거시지표로 현재 국면을 과거와 유사도 매칭(퀀트 접근).
- **상세**: FRED 매크로 시계열 수집(※ FRED API/시리즈 직접 호출 차단 → 대체 수집 방식 확정 필요) → 국면 분류·현재 위치 표시. ETF 관점 국면별 가이드.
- **선행 과제**: 수집 방식 확정 + "의미 있는 지표 목록" 정의.
- **수용 기준**: 현재 매크로 국면 표시 + 과거 유사 국면 참조.

### 9. 애널리스트 리포트 데이터 (매수 추천 N명)
- **라벨**: new-feature, data / P2 / L
- **목적**: "10명 중 N명 매수 추천"식 컨센서스 표시(토스증권식).
- **상세**: 애널리스트 컨센서스 데이터 소스 확보(Seeking Alpha 등) → 종목별 매수/중립/매도 분포 표시. 추천 후 주가 추이 연계.
- **수용 기준**: 종목 상세에 애널리스트 컨센서스 표시.

### 10. 애널리스트·기자별 예측 승률 추적
- **라벨**: new-feature, data / P3 / L
- **목적**: 누가 잘 맞추는지 적중률 표시(TipRanks식).
- **상세**: 애널리스트/기자 예측 이력 누적 → 적중 여부 추적·승률 계산. 데이터 소스/추적 파이프라인 필요.
- **수용 기준**: 주요 애널리스트·기자별 승률 지표 제공.
- **주의**: 데이터 확보·정확도 검증 선행. (일부는 "고민 건"에 가까움)

### 11. 인용 출처 수집·역추적 ("~에 따르면" 마이닝)
- **라벨**: new-feature, data / P3 / L
- **목적**: 리포트/뉴스의 공통 출처를 대량 수집 → 승률 높은 기자가 공통으로 보는 소스 발견(차별화 인사이트).
- **상세**: "~에 따르면/출처" 패턴 스크래핑·집계 → 빈도 분석. 승률 데이터와 매칭. 단계적: 우선 출처 수집·집계만.
- **수용 기준**: 출처 수집·집계 결과 확인 가능.
- **주의**: 가설 검증형 — 1단계(수집·집계)만 이슈화, 추론 단계는 별도.

### 12. 신흥국(인도) 시장 지표 추가
- **라벨**: new-feature, data / P2 / L
- **목적**: 인도 등 국가별 지표 누적·표시(차별화 콘텐츠). ※ 비즈니스 게이팅(유료/회원 개방)은 별도 고민 건.
- **상세**: 국가별 지표 수집·누적 → 상승세/고점 판단 가능한 표·차트. 시장 범위 정책(현 미국·한국 한정)과 충돌하므로 데이터 모델 영향 큼.
- **수용 기준**: 인도 국가 지표 페이지/표 제공.
- **주의**: `sk-orchestrator` 하네스 대상(데이터 모델+필터+API+UI 동시 영향). 단독 이슈보다 기획서 선행 권장.

## D. 콘텐츠·배포 자동화

### 13. 모닝/이브닝 브리핑 + TTS 팟캐스트
- **라벨**: new-feature, content / P2 / L
- **목적**: 아침=밤사이 미장 요약, 퇴근=한국장 요약. TTS 음성 변환 → 팟캐스트/유튜브.
- **상세**: 시간대별 요약 생성 → 구글 TTS로 음성화 → 메인에 라디오 위젯 + 유튜브 링크. 참고 소스(매크로 뉴스) 지정 필요.
- **수용 기준**: 1일 2회 브리핑 생성·재생, 음성 출력.

### 14. 주말 브리핑 ("월요일 출근 전 봐야 할 2가지")
- **라벨**: new-feature, content / P2 / M
- **목적**: 금요일 장 마감 후 쏟아지는 뉴스 + 주말 이슈 정리. 월요일 대비 핵심 2~3개 제시.
- **상세**: 주말 이슈 수집·요약 → 일요일 저녁 알림. "주목할 것" 위주.
- **수용 기준**: 주말 브리핑 생성 + 일요일 저녁 알림.

### 15. 주간/월간 종합 리포트
- **라벨**: new-feature, content / P2 / M
- **목적**: 토·일 뉴스 적음 → 주간 종합. 월말 트렌드 리포트.
- **상세**: 누적 일간 데이터를 주/월 단위로 집계·트렌드 추출.
- **수용 기준**: 주간·월간 리포트 자동 생성.

### 16. 섹터킹 텔레그램 자동 포스팅
- **라벨**: new-feature, content / P3 / M
- **목적**: 매크로 속보를 텔레그램 채널에 매일 자동 발행(유입 채널). 쓰레드 자동 포스팅과 동일 패턴.
- **상세**: 브리핑/속보를 텔레그램 봇으로 자동 전송. 사이트 링크 포함.
- **수용 기준**: 채널 자동 포스팅 동작.

## 제외 — "먼저 고민해야 할 건" (이슈 등록 보류)

코드 작업 전에 방향·정책 결정이 먼저 필요한 항목:

- **타겟·포지셔닝**: 누가 쓸 것인가(중소형 운용사 vs 개인 vs 금융권 실무자). 모든 기능의 전제.
- **수익화 모델**: 무료배포 침투→할인쿠폰 바이럴, 구독(토큰) 모델, 유튜브 광고, 광고 슬롯 판매 — 비즈니스 의사결정.
- **인도/신흥국 게이팅 정책**: 가격(월 5,500원), 회원 활동 보상 개방 — 기능(#12)과 분리된 정책.
- **고래(큰손) 거래 추적**: 주식은 개인 단위 데이터 확보 불확실(코인만 가능) → 데이터 가용성 검토 먼저.
- **승률 추적(#10)·소스 역추적(#11)**: 가설 검증형이라 데이터 확보·정확도 선검증 필요(일부만 이슈화).
- **텔레그램 방 '운영' 전략**: 자동 포스팅(#16)은 기능이나, 콘텐츠 톤·리딩방 차별화는 운영 고민.
- **네이밍**: "섹터킹" 유지로 결정 완료.
