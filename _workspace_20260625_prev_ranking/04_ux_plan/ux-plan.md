# Sector King — IA 재설계 및 사용자 동선 기획서

> 작성: `sk-ui-planner`
> 일자: 2026-05-09
> 범위: 전체 페이지 IA·동선 재설계 (디자인 디테일 제외 → `sk-design-system`에 위임)
> 전제: region 토글(전체/국내/해외) 도입은 이미 결정됨 (`integrated-plan.md`). 본 문서의 IA에 동일하게 반영.

---

## 0. 한눈 요약 (TL;DR)

1. **메인은 "오늘의 시장 한 페이지"로 재구성** — 헤로 KPI 띠 + 자금 흐름 + 산업 카드(미니 인사이트 포함) 순.
2. **루트 레벨 `/money-flow`, `/price-changes`, `/statistics` 3개를 폐기** — 산업 컨텍스트 없는 동일 분석은 사용자 가치가 낮고, 이미 `/industry-money-flow`(전 산업 비교)와 메인이 그 역할을 한다.
3. **`/industry-money-flow`는 `/money-flow`로 이전(canonical)** — 의미가 명확한 짧은 URL. 기존 `/industry-money-flow`는 301.
4. **산업 컨텍스트 페이지는 4-탭(패권 지도 / 자금 흐름 / 등락율 / 통계)으로 통일** — 헤더에 산업명 + 탭 + region/period/날짜.
5. **종목(회사) 상세는 모달 유지** — 단, 모든 페이지에서 동일 `<CompanyDetail />` 모달이 열리고, 모달 내부에 "이 회사가 속한 섹터·카테고리·산업으로 이동" 링크 추가 (역방향 드릴업 동선 확보).
6. **글로벌 네비**(상단 바) 신설 — 메인·전 산업 자금 흐름·산업 선택 드롭다운·방법론 + 검색·region·테마.
7. **빵부스러기(브레드크럼)** 산업 페이지·하위 탭에 표준 적용 — `메인 / 반도체 / 자금 흐름`.

## 1. 페이지 인벤토리 + 진단

| 경로 | 현재 역할 (1줄) | 핵심 인사이트 | 진단 — 문제점 | 제안 |
|------|------|------|------|------|
| `/` | 산업 카드 그리드 + 산업별 자금 흐름 카드 + 회사 통계/등락율 카드 | "오늘 어느 산업이 뜨거운가" | 헤로(시장 요약) 부재. 카드가 단순 카운트만 노출 → 인사이트 없음. 진입 후 어디로 갈지 모호 | **Refactor** — 시장 KPI 띠 + 카드당 미니 인사이트 + CTA 명확화 |
| `/[industryId]` | 산업 패권 지도 (카테고리×섹터 그리드) | "이 산업의 구조와 시장 지배 종목" | 가장 정보량이 많은 페이지인데 진입 직후 의도 파악 어려움. "왜 봐야 하는지" 헤더 카피 부재 | **Keep + Tab 컨테이너 통합** |
| `/[industryId]/money-flow` | 산업 내 섹터별 자금 유입/유출 | "이 산업에서 돈이 들어가는 섹터" | 헤더 분기 링크 3개(전체/자금/등락) 있으나 "통계"는 빠져 있음. 일관성 결여 | **탭 4개로 통일** |
| `/[industryId]/price-changes` | 산업 내 종목 등락율 | "이 산업 종목의 단기 모멘텀" | 동일 (탭 일관성 부재) | **탭 4개로 통일** |
| `/[industryId]/statistics` | 산업 내 회사·섹터 트렌드 | "이 산업 종목의 장기 성장·등장 빈도" | "회사 등장 통계"라는 메타데이터 라벨이 사용자에게 모호 | **이름 변경 → "기업·섹터 트렌드"** + 탭 통일 |
| `/industry-money-flow` | 전 산업 자금 흐름 비교 | "오늘 시장에서 가장 큰 자금 이동은 어느 산업?" | URL이 길고 위치 모호. 메인에도 동일 카드 있음 | **`/money-flow`로 이전(canonical)** + 메인 카드는 요약 인사이트 모듈로 |
| `/money-flow` (루트) | 모든 산업 합집합 자금 흐름 (legacy) | "산업 무관 전체 자금 이동" | `/industry-money-flow`와 거의 중복. 산업 컨텍스트 없으면 액션 가능성 낮음 | **폐기 → `/money-flow`(=구 industry-money-flow)로 301** |
| `/price-changes` (루트) | 전 종목 등락율 | "오늘 가장 오른/내린 종목" | 산업 컨텍스트 없어 너무 광범위. 검색·메인 추천으로 대체 가능 | **폐기 → 메인의 "오늘의 핫 종목" 모듈로 흡수** |
| `/statistics` (루트) | 전 종목 통계 | "전 시장 기업 분포" | 거의 사용되지 않을 화면. 메인·산업 통계로 충분 | **폐기** |
| `/methodology` | 방법론 문서 | "데이터·점수의 신뢰 근거" | 내용 양호. 푸터·헤더에 진입점만 보강 필요 | **Keep** (푸터 링크, 카드 미니 KPI 옆 "?" 툴팁 추가) |
| `/design-system` | 디자인 토큰 카탈로그 (개발용) | (개발 전용) | 외부 노출 X | **Keep** (robots disallow 확인) |

### 1.1 동선 끊김(dead-end) 식별

- 종목 상세 모달에서 "이 종목이 속한 섹터로 이동" 링크 부재 → 사용자는 모달 닫고 다시 산업 페이지에서 섹터 찾아야 함.
- `/[industryId]/money-flow`에서 섹터 카드 클릭 시 그 섹터의 종목 리스트는 보이나, 그 섹터의 "통계 탭"으로 점프 불가 → 한 섹터를 깊게 보려면 탭 왕복 발생.
- 메인 산업 카드 → 패권 지도로 진입하지만, 패권 지도의 섹터 카드 → 자금 흐름 등 하위 분석으로의 cross-link 없음.
- 검색에서 회사 선택 → 모달은 열리지만 "관련 회사·동일 섹터 탑5" 추천 부재.

### 1.2 중복 정보

- `/industry-money-flow`(전 산업 비교) ↔ 메인 `IndustryMoneyFlowCard` ↔ `/money-flow` 3중 중복.
- 패권 지도 사이드바의 `<CompanyStatistics />` ↔ `/[industryId]/statistics` 컨텐츠 중복(요약 vs 전체).

### 1.3 모호한 진입점

- 메인에서 "산업별 자금 흐름" 섹션이 산업 카드 그리드보다 위에 있어, 사용자가 첫 산업 진입 동선을 잡기 어려움. 헤로 → 산업 그리드 → 디테일 흐름이 자연스러움.
- 산업 페이지 헤더의 자금/등락 버튼은 있는데 "통계"는 없음 → 통계 탭 진입 동선이 사실상 차단되어 있음.

## 2. 페르소나 + Job-to-be-Done

### 2.1 페르소나 A — "산업 트렌드 스캐너" (전략 분석가)
- **Job:** 매일 아침 5분 안에 "오늘 시장에서 무슨 일이 있었는지" 파악.
- **핵심 질문:**
  - 오늘 시장 전체 시총 변화는?
  - 가장 자금이 몰린 산업·섹터는?
  - 가장 큰 등락 종목은?
- **선호 동선:** 메인 헤로 → 산업 자금 흐름 비교 → (필요 시) 특정 산업 패권 지도.
- **체류:** 메인 + `/money-flow` (전 산업 비교).

### 2.2 페르소나 B — "섹터·종목 평가자" (개인 투자자)
- **Job:** 특정 산업·섹터의 종목 후보를 비교·평가.
- **핵심 질문:**
  - AI 인프라에서 자금이 가장 많이 들어오는 회사는?
  - 반도체에서 등락률이 가장 큰 회사·섹터는?
  - 이 회사의 점수·재무 지표는?
- **선호 동선:** 메인 → 산업 카드 → 패권 지도 → 섹터 클릭 → 회사 모달 → (필요 시) 자금/등락/통계 탭으로 같은 산업 다른 시각.
- **체류:** `/[industryId]` + 4개 하위 탭.

### 2.3 페르소나 C — "신뢰성 검증자" (데이터 신중파)
- **Job:** 사이트의 점수·데이터 출처가 믿을 만한지 확인.
- **핵심 질문:**
  - 점수 산식은 무엇인가?
  - 데이터 갱신 주기는?
  - 한국 주식의 환율 처리 방식은?
- **선호 동선:** 어디서든 "?" 툴팁 → `/methodology`.
- **체류:** `/methodology`.

> 본 IA는 페르소나 A·B를 1차 타깃으로 하며, C에는 진입점 보강만 한다.

## 3. 새로운 IA 트리

```
/                                        [메인 — 시장 한 페이지]
├── /[industryId]                        [산업 컨텍스트 — 4탭 통합 컨테이너]
│   ├── tab=map (default)                · 패권 지도
│   ├── tab=flow                         · 자금 흐름
│   ├── tab=price                        · 등락율
│   └── tab=stats                        · 기업·섹터 트렌드 (구 statistics)
│
│   * 또는 라우트로 분리(현행 유지):
│     /[industryId]/money-flow
│     /[industryId]/price-changes
│     /[industryId]/statistics
│   → 본 기획에서는 라우트 분리는 유지하되 시각적으로 하나의 탭 컨테이너로 묶는다.
│
├── /money-flow                          [전 산업 자금 흐름 비교 — 구 /industry-money-flow]
├── /methodology                         [방법론]
├── /design-system                       [개발 전용]
└── (모달) <CompanyDetail ticker=…/>      [어디서나 호출 가능]
```

### 3.1 폐기되는 라우트 (301 매핑)

| 폐기 경로 | 리다이렉트 대상 | 비고 |
|---|---|---|
| `/industry-money-flow` | `/money-flow` | canonical 짧은 URL로 통일 |
| `/money-flow` (legacy 전체) | `/money-flow` (구 industry-money-flow) | 의미 통일 |
| `/price-changes` (루트) | `/` (메인의 핫 종목 모듈) | 사용 가치 낮음 |
| `/statistics` (루트) | `/` | 동일 |

> `next.config.ts`의 redirects에서 처리. 검색 색인 갱신.

### 3.2 동선 다이어그램 (텍스트)

```
[메인 /] ──헤로 KPI 띠──┐
   │                    │
   │   [산업 자금 흐름   │
   │    비교 카드]       │ ──"전체 보기"──▶ [/money-flow 전 산업 비교]
   │                    │
   │   [산업 카드 그리드 — 카드당 미니 인사이트]
   │        │
   │        └──카드 클릭──▶ [/[industryId] 패권 지도]
   │                            │
   │                            ├──섹터 카드──▶ [섹터 종목 리스트 인-페이지]
   │                            │                    │
   │                            │                    └──종목 클릭──▶ [<CompanyDetail> 모달]
   │                            │                                       │
   │                            │                                       ├─"섹터로 이동"
   │                            │                                       ├─"카테고리로 이동"
   │                            │                                       └─"산업 자금흐름에서 보기"
   │                            │
   │                            └──탭──▶ [자금흐름 / 등락 / 통계]
   │                                          │
   │                                          └──섹터/종목 클릭──▶ <CompanyDetail> 모달
   │
   ├──"오늘의 핫 종목" 모듈──▶ <CompanyDetail> 모달
   │
   └──검색──▶ 종목 결과 클릭──▶ <CompanyDetail> 모달
```

핵심: **드릴다운(메인→산업→섹터→종목)** 일방향 + **드릴업(종목 모달 → 섹터/카테고리/산업)** 역방향이 모두 1클릭.

## 4. 페이지별 변경안 (현재 → 제안)

### 4.1 메인 `/`

| 항목 | 현재 | 제안 |
|------|------|------|
| 헤더 | 로고·timestamp·region·share·search·help·theme | 추가: **글로벌 네비**(메인·자금흐름·산업▼·방법론) |
| 1번 섹션 | "산업별 자금 흐름" (위) | **헤로 KPI 띠** — 전체 시총·전일대비·핫 산업 1·핫 섹터 1·자금 이동 1위 산업 |
| 2번 섹션 | 산업 카드 그리드 | **유지하되 카드당 인사이트 1줄** (예: "AI 인프라 — 자금유입 1위 NVDA") |
| 3번 섹션 | "시장 동향 요약" 회사 통계+등락율 | "**오늘의 핫 종목**" 통합 모듈 (등락 상위 5 + 자금유입 상위 5 좌우 분할) |
| 4번 섹션(신규) | — | **산업 자금 흐름 비교 카드** — 한눈 비교 + "전체 보기" CTA → `/money-flow` |
| 푸터(신규) | — | 방법론·면책·갱신시각·소셜 |

### 4.2 산업 컨텍스트 `/[industryId]`

| 항목 | 현재 | 제안 |
|------|------|------|
| 헤더 | "Sector King" 로고 + 산업명 + 4개 헤더 링크 (전체/자금/등락) | **글로벌 네비 + 산업 컨텍스트 바**(브레드크럼: 메인 / 반도체 + 4탭: 패권지도·자금흐름·등락율·기업트렌드) |
| 탭 진입 | 헤더 링크 (불일치, 통계 누락) | 4탭 일관 — 패권지도(default) / 자금흐름 / 등락율 / 기업트렌드 |
| 콘텐츠 | 카테고리 그리드 + 사이드바(통계+등락) | 그대로. 단 사이드바는 **"이 산업의 핵심 회사"** 모듈로 리네이밍 + "더보기" → 통계 탭 |
| 빈 region 처리 | 카테고리 숨김 (already in plan) | 유지 |

| 헤더 | 자체 | 글로벌 네비 + 페이지 타이틀 "산업별 자금 흐름 비교" + period + region |
| 콘텐츠 | 산업 비교 시각화 | 유지 |
| CTA | — | 산업 카드 클릭 시 → `/[industryId]?tab=flow` |

### 4.7 `/methodology`

| 항목 | 현재 | 제안 |
|------|------|------|
| 헤더 | 단순 뒤로가기 | 글로벌 네비 + 목차(현행 유지) |
| 진입 동선 | 알 수 없음 | **메인 푸터 + 카드 인사이트 옆 "?" 툴팁 → 해당 섹션으로 anchor 점프** |


## 8. sk-design-system과의 협업 인터페이스

### 8.1 디자인 시스템에 위임할 항목

| 항목 | 요청 내용 |
|------|---------|
| **Hero KPI 카드** | 4 KPI 카드 컴포넌트 (라벨·값·증감 색상·툴팁 진입점) |
| **산업 카드 (인사이트 포함)** | 카드 안의 인사이트 1줄 행 디자인 (icon + 텍스트 + 강조 칩) |
| **글로벌 네비 (top bar)** | 메뉴 항목 hover/active 상태, 산업 드롭다운 |
| **산업 컨텍스트 바 (4탭)** | 탭 컴포넌트 (활성 강조, 모바일 가로 스크롤, 브레드크럼과의 시각 위계) |
| **CompanyDetail 모달 cross-link 영역** | 모달 하단 "이 섹터로 이동" 등 3개 액션 행 디자인 |
| **빈 상태 (`<EmptyRegionState>`)** | 이미 정의됨 (`ui-plan.md` §5.2) — 통일 적용 |
| **푸터** | 글로벌 푸터 |
| **모바일 메뉴(햄버거)** | 글로벌 네비의 모바일 변형 |

### 8.2 sk-design-system이 결정할 디테일

- 색상 토큰 (KPI 증감, 산업 카드 강조)
- 타이포그래피 (헤로 숫자, 인사이트 카피)
- 아이콘 매핑 (lucide 사용 — 이모지 절대 금지 규칙 준수)
- 다크모드 검수
- 브레이크포인트별 그리드 (헤로 4열 / mobile 2×2)

### 8.3 본 IA가 보존하는 제약

- region 토글 위치 — 기존 합의(`ui-plan.md`) 그대로
- 반응형 헤더 표준 패턴 — `flex flex-col sm:flex-row sm:items-center sm:justify-between gap-3`
- 이모지 금지·lucide-react 아이콘
- URL 파라미터 (UUID·numeric ID 사용 규칙)

### 8.4 다른 에이전트에 보내는 시그널

- `sk-filter-architect`: 변경 없음 (region 합의 유지).
- `sk-data-modeler`: 미니 인사이트(예: 산업 카드의 "자금유입 1위 회사")는 기존 API로 계산 가능 — 신규 컬럼 불필요. `industries` 응답에 산업당 top1 회사·섹터 추가 (선택적 필드).
- `sk-implementer`: layout 통합·라우트 redirects·신규 컴포넌트 3개(`global-nav`, `industry-context-bar`, `hero-kpi`, `hot-tickers`).
- `sk-verifier`: 폐기 라우트 301 검증, 4탭 a11y(`aria-current="page"`, `role="tablist"` 또는 nav 패턴), 모달 cross-link 키보드 탐색 검증.
